# 🤖 AI Mocker — User Guide

Prism's **AI Mocker** replaces the default faker-based response generator with an LLM-powered pipeline that produces context-aware, schema-compliant API responses. The mock server *remembers* previous interactions, so a `POST` followed by a `GET` returns logically consistent data — just like a real backend.

---

## Quickstart

### 1. Set Environment Variables

The AI Mocker needs access to a chat model and an embedding model. Configure one of the providers below.

#### OpenAI (default)

```bash
export OPENAI_API_KEY="sk-..."
```

#### Azure OpenAI

If your URL contains `.openai.azure.com`, Prism auto-detects Azure and parses the deployment details from the URL — no extra config needed.

```bash
export OPENAI_API_URL="https://<instance>.openai.azure.com/openai/deployments/<chat-deployment>/chat/completions?api-version=2025-01-01-preview"
export OPENAI_API_KEY="<your-key>"

export EMBEDDING_API_URL="https://<instance>.openai.azure.com/openai/deployments/<embedding-deployment>/embeddings?api-version=2023-05-15"
export EMBEDDING_API_KEY="<your-key>"   # falls back to OPENAI_API_KEY if unset
```

### 2. Launch Prism with `--ai`

```bash
prism mock your-spec.yaml --ai
```

That's it. Every response is now generated by the LLM, validated against the OpenAPI schema, and stored in a local SQLite memory database.

### 3. Try It Out

```bash
# Create an order
curl -s -X POST http://localhost:4010/store/order \
  -H "Content-Type: application/json" \
  -d '{"id": 7, "petId": 999, "quantity": 5, "status": "approved"}' | jq

# Retrieve it — the AI remembers the POST
curl -s http://localhost:4010/store/order/7 | jq
```

## CLI Options

| Flag | Description |
|---|---|
| `--ai` | Enable AI-powered response generation (requires LLM credentials) |
| `--port <n>` | Server port (default: `4010`) |
| `--host <addr>` | Bind address (default: `127.0.0.1`; use `0.0.0.0` for Docker) |
| `--dynamic` | Use dynamic faker generation as the fallback strategy |

All standard Prism flags (`--cors`, `--errors`, `--verboseLevel`, etc.) work alongside `--ai`.

## Environment Variable Reference

### Chat Model

| Variable | Description | Default |
|---|---|---|
| `OPENAI_API_URL` | Full URL for the chat completions endpoint. Azure URLs are auto-detected. | OpenAI default |
| `OPENAI_API_KEY` | API key for the chat model | — |
| `AI_PROVIDER` | Explicit provider override: `openai` or `azure` | `openai` |
| `AI_MODEL` | Override the model name | `gpt-4o-mini` |

### Embedding Model

| Variable | Description | Default |
|---|---|---|
| `EMBEDDING_API_URL` | Full URL for the embeddings endpoint. Azure URLs are auto-detected. | OpenAI default |
| `EMBEDDING_API_KEY` | API key for embeddings (falls back to `OPENAI_API_KEY`) | — |
| `EMBEDDING_PROVIDER` | Explicit provider override: `openai` | `openai` |
| `EMBEDDING_MODEL` | Override the embedding model name | `text-embedding-3-small` |

### Memory & Runtime

| Variable | Description | Default |
|---|---|---|
| `PRISM_AI_DB_PATH` | Path to the SQLite memory database file | `.prism-memory.db` |
| `PRISM_AI_DISABLE_VEC` | Set to `true` to disable the `sqlite-vec` extension | `false` |

## How It Works

```
Request → Context Agent → Generator Agent → Memory Agent → Response
              │                  │                │
        Retrieve past       LLM generates     Persist this
        interactions        schema-compliant   interaction for
        from memory         response           future context
```

1. **Context Agent** — summarizes the incoming request, embeds it, and retrieves relevant past interactions from the local SQLite vector store.
2. **Generator Agent** — sends the schema + context + request to the LLM via `withStructuredOutput`, then validates the response with Ajv.
3. **Memory Agent** — stores the request/response pair as an embedding so future requests can reference it.

### Caching & Concurrency

- **Response Cache**: Identical repeated `GET` requests return cached LLM responses without re-calling the model.
- **Resource Mutex**: Concurrent mutations to the same resource path are serialized.
- **LLM Concurrency Limiter**: At most 5 parallel LLM calls at any time.
- **Timeouts**: Embedding (3s), LLM generation (15s), and full pipeline (20s) each have independent budgets. On timeout, the system gracefully falls back to faker.

### Graceful Fallback

If the LLM is unavailable, keys are missing, or any stage of the pipeline fails, Prism seamlessly falls back to its standard faker-based generation. You'll see a warning in the logs but the server never crashes.

## Resetting the Memory Database

The AI Mocker stores all interactions in a local SQLite database (`.prism-memory.db` by default). Over time, this memory influences future responses.

To start fresh:

```bash
rm -f .prism-memory.db .prism-memory.db-shm .prism-memory.db-wal
```

> **When to reset**: If you've been running with broken configuration (missing keys, timeouts) and the memory has captured fallback/faker responses, the LLM may mimic those values in future runs. Wiping the database lets the AI generate from scratch.

You can also point to a different database per session:

```bash
PRISM_AI_DB_PATH=./my-session.db prism mock spec.yaml --ai
```

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| Responses look like `-9007199254740991` | AI pipeline is falling back to faker | Check logs for errors; verify API keys are set |
| `"LLM generation threw an error: strict mode: unknown keyword"` | OpenAPI schema contains extensions (`xml`, `example`) | Already handled internally; update to latest build |
| `"embedding timed out after 3000ms"` | Slow network or cold-start on embedding model | Retry; first call is always slowest |
| Server crashes on startup | API key validation fails synchronously | Ensure keys are exported in the current shell |
| AI returns data matching previous faker responses | Stale memory database from broken runs | Reset the database (see above) |
| `EADDRINUSE` error | Another Prism instance is running on the same port | `kill $(lsof -t -i:<port>)` or use a different `--port` |
